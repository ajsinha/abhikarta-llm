Introduction to Abhikarta Workflows

What is Abhikarta?

Abhikarta is a comprehensive LLM execution platform that enables you to build sophisticated AI-powered workflows. It combines:

  • Chat Interfaces - Interactive AI conversations
  • RAG (Retrieval-Augmented Generation) - Knowledge-enhanced AI responses
  • Workflow Orchestration - Complex multi-step AI processes
  • DAG Processing - Directed acyclic graph execution
  • Autonomous Planning - AI-driven decision making
  • Human-in-the-Loop - Human oversight and approval

Workflow Architecture

Abhikarta workflows are built on top of LangChain and LangGraph, providing a robust foundation for AI orchestration. The system supports multiple execution patterns:

Sequential

Execute nodes one after another in a linear flow

Parallel

Run multiple branches simultaneously for faster processing

Conditional

Branch execution based on dynamic conditions

Human-in-the-Loop

Pause for human approval before proceeding

Core Components

Component Description Use Cases
Workflow Engine LangGraph-based execution engine Process workflows, manage state, handle errors
Node Types Building blocks for workflows LLM, Tool, Human, Conditional, Parallel
State Management SQLite-based persistence Resume workflows, audit trails, checkpointing
Database Layer 6-table schema for workflow data Workflows, executions, tasks, variables, audit logs
API Layer Flask routes for workflow management Create, execute, monitor, control workflows

Workflow Definition Format

JSON Structure

{
  "nodes": [
    {
      "node_id": "unique_identifier",
      "node_type": "llm|tool|human|conditional|parallel",
      "config": {
        // Node-specific configuration
      }
    }
  ],
  "edges": [
    {
      "source": "source_node_id",
      "target": "target_node_id"
    }
  ]
}

Python Structure

from workflow_management.workflow_langgraph_engine import LangGraphWorkflowEngine

# Create workflow
workflow = {
    'nodes': [
        {'node_id': 'start', 'node_type': 'start'},
        {'node_id': 'process', 'node_type': 'llm', 'config': {...}},
        {'node_id': 'end', 'node_type': 'end'}
    ],
    'edges': [
        {'source': 'start', 'target': 'process'},
        {'source': 'process', 'target': 'end'}
    ]
}

# Execute
engine = LangGraphWorkflowEngine(workflow)
result = engine.execute({'input': 'your data'})

Sequential Workflows

What are Sequential Workflows?

Sequential workflows execute nodes one after another in a linear chain. Each node processes the output of the previous node. This is the simplest and most common workflow pattern.

Use Cases

  • Text Processing Pipeline - Extract → Analyze → Summarize → Generate
  • Data Transformation - Validate → Clean → Transform → Store
  • Content Generation - Research → Outline → Write → Edit
  • Customer Support - Classify → Route → Respond → Log

Execution Flow

┌───────┐      ┌──────────┐      ┌──────────┐      ┌──────────┐      ┌─────┐
│ Start │  →   │  Node 1  │  →   │  Node 2  │  →   │  Node 3  │  →   │ End │
└───────┘      └──────────┘      └──────────┘      └──────────┘      └─────┘
               Extract Text      Analyze         Summarize

Best Practices for Sequential Workflows
  • Clear Node Names: Use descriptive IDs like "extract_text" not "node1"
  • State Passing: Each node can access previous results via {previous_result}
  • Error Handling: Use try-catch in tool nodes to handle failures gracefully
  • Temperature Control: Lower temperature (0.2-0.3) for factual tasks, higher (0.7-0.9) for creative tasks
  • Prompt Engineering: Be specific about expected output format

Parallel Workflows

What are Parallel Workflows?

Parallel workflows execute multiple branches simultaneously, allowing independent tasks to run concurrently. This dramatically reduces overall execution time when tasks don't depend on each other.

Use Cases

  • Multi-Model Analysis - Run different AI models simultaneously on the same input
  • Data Processing - Process multiple data sources concurrently
  • Content Generation - Generate multiple variations in parallel
  • Research Aggregation - Query multiple sources simultaneously

Execution Flow

                     ┌──→ Branch 1 (GPT-4 Analysis) ──┐
                     │                                 │
┌───────┐      ┌─────────┐                       ┌─────────┐      ┌─────┐
│ Start │  →   │ Parallel│                       │ Combine │  →   │ End │
└───────┘      └─────────┘                       └─────────┘      └─────┘
                     │                                 │
                     └──→ Branch 2 (Claude Analysis) ─┘

All branches run simultaneously
Results merged at combine node

Conditional Workflows

What are Conditional Workflows?

Conditional workflows make decisions during execution, routing to different branches based on runtime conditions. This enables dynamic, context-aware processing.

Use Cases

  • Priority Routing - High-priority items take a fast-track path
  • Quality Gates - Only high-quality outputs proceed to next stage
  • Error Recovery - Different handling for errors vs. success
  • User Segmentation - Different flows for different user types

Execution Flow

                              ┌──→ High Priority Path ──→ Escalate ──┐
                              │                                      │
┌───────┐      ┌──────────┐  │                                      │    ┌─────┐
│ Start │  →   │ Analyze  │  → [Conditional: Priority > 0.8?]       →  →│ End │
└───────┘      └──────────┘  │                                      │    └─────┘
                              │                                      │
                              └──→ Normal Path ──→ Standard Process ─┘

Decision made at runtime based on analysis results

Conditional Logic Best Practices
  • Clear Conditions: Use simple, testable conditions like "score > 0.8"
  • Boolean Expressions: Combine with AND/OR: "urgency > 0.7 or sentiment < -0.5"
  • State Access: Reference result fields: result['field_name']
  • Default Paths: Always have a fallback for when condition is false
  • Testing: Test both true and false paths thoroughly

Human-in-the-Loop (HITL) Workflows

What are HITL Workflows?

Human-in-the-Loop workflows pause execution at critical points to wait for human approval, input, or decision. This ensures human oversight in automated processes, combining AI efficiency with human judgment.

Use Cases

  • Content Approval - Review AI-generated content before publishing
  • Financial Decisions - Approve large transactions or investments
  • Quality Control - Verify AI outputs meet standards
  • Policy Compliance - Ensure actions comply with regulations
  • Critical Operations - Human verification for important actions

Execution Flow

┌───────┐    ┌────────┐    ┌────────────────┐    ┌──────────┐    ┌─────┐
│ Start │ → │Generate│ → │ Human Approval │ → │ Publish  │ → │ End │
└───────┘    └────────┘    └────────────────┘    └──────────┘    └─────┘
                                  ↓
                            ⏸ PAUSED
                            Waiting for human
                            Task created in UI
                            Email notification sent

HITL Workflow Behavior

When workflow reaches human node:

  1. Workflow execution PAUSES (status='paused')
  2. Human task is created in database
  3. Task appears in assigned user's task list
  4. Email notification sent (if configured)
  5. Workflow waits indefinitely (or until timeout)
  6. User completes task through UI
  7. Workflow RESUMES automatically
  8. Proceeds to next node based on user's decision
Critical HITL Configuration
Parameter Required Description Example
assigned_to ✅ Yes User email or ID manager@company.com
task_type ✅ Yes approval/review/input approval
instructions ✅ Yes What user should do Review and approve report
timeout ⚠️ Recommended Seconds before timeout 3600 (1 hour)
allow_comments ❌ Optional Can user add feedback? true
attachments ❌ Optional Files to show user ["report.pdf"]

# Define custom reducer
from typing import Annotated
import operator

def intelligent_merge(results: list) -> dict:
    """Merge parallel results with conflict resolution"""
    if all(r.get('sentiment') == 'positive' for r in results):
        return {'consensus': 'positive', 'confidence': 0.95}
    elif all(r.get('sentiment') == 'negative' for r in results):
        return {'consensus': 'negative', 'confidence': 0.95}
    else:
        return {'consensus': 'mixed', 'confidence': 0.6}

# Use in workflow state
class WorkflowState(TypedDict):
    results: Annotated[list, intelligent_merge]
