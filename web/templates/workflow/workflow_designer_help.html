Getting Started

What is the Visual Workflow Designer?

The Visual Workflow Designer is a drag-and-drop interface that lets you create complex AI-powered workflows without writing any code. You can combine AI models, tools, human approvals, and logic to build sophisticated automation.

Your First Workflow in 5 Steps

1 Access the Designer

Go to Workflows → Visual Designer in the menu

2 Drag Nodes onto Canvas

From the left palette, drag these nodes onto the canvas:

  • Start - Entry point (required)
  • LLM - AI processing
  • End - Exit point (required)
3 Connect the Nodes

Click the blue dot on the right of Start, drag to the blue dot on the left of LLM. Repeat to connect LLM to End.

[Start] ───→ [LLM] ───→ [End]
4 Configure the LLM Node

Click the LLM node to open properties panel on the right:

  • Node ID: analyze_text
  • Model: GPT-4
  • Prompt: Analyze the sentiment of: {input}
  • Temperature: 0.7

Click "Save Properties"

5 Save Your Workflow

Click the Save Workflow button in the toolbar, enter a name, and save!

Success! You've created your first workflow. It will analyze text sentiment using GPT-4.

Saving & Loading Workflows

Save Workflow

Save your workflow to the database so you can use it later:

1 Click Save Workflow Button

Click the Save Workflow button in the toolbar

2 Enter Workflow Name

Give your workflow a descriptive name (e.g., "Sentiment Analysis Pipeline")

3 Confirm Save

Click the save button in the modal - your workflow is now saved!

💡 Tip: Use the Validate button before saving to check for errors!

Load Workflow from File

Load a workflow from a JSON file on your computer:

1 Click Load Button

Click the Load button in the toolbar

2 Select JSON File

A file dialog will open - select your workflow .json file

3 Workflow Loads Automatically

The workflow will be loaded onto the canvas with all nodes and connections

✅ Best Practice: Load works with any valid workflow JSON file, including those exported from this designer!

Export JSON

Export your current workflow as a JSON file:

1 Click Export JSON Button

Click the Export JSON button in the toolbar

2 Choose Export Method

You have two options:

  • Copy to Clipboard - Instantly copy the JSON to paste elsewhere
  • Download as File - Save as a .json file with timestamp
💡 Use Cases for Export:
  • Share workflows with team members
  • Version control (commit to Git)
  • Backup important workflows
  • Migrate between systems
  • Create workflow templates

Keyboard Shortcuts

Shortcut Action
Ctrl + S Save workflow
Note: The "Import JSON" (paste) feature has been removed due to stability issues. Use the Load button (file dialog) instead - it's more reliable and provides a better user experience.

Node Types Reference

Start Node

Purpose: Entry point for every workflow

Requirements: Every workflow must have exactly ONE start node

Properties: None

Connections: Output only (can connect to other nodes)

Tip: Place the Start node on the left side of your canvas to establish clear workflow direction.

End Node

Purpose: Exit point for workflow

Requirements: Every workflow must have at least ONE end node

Properties: None

Connections: Input only (receives from other nodes)

Tip: You can have multiple End nodes for different outcomes (success, error, etc.)

LLM Node

Purpose: Call an AI language model (GPT-4, Claude, etc.)

Use Cases: Text analysis, generation, summarization, translation

Properties:
Property Description Example
Model Which AI model to use GPT-4, GPT-3.5, Claude-3
Prompt Template Instructions for the AI Summarize this text: {input}
Temperature Creativity level (0-2) 0.7 (balanced), 0.3 (focused), 1.5 (creative) 
{
  "node_id": "analyze",
  "node_type": "llm",
  "config": {
    "model": "gpt-4",
    "prompt_template": "Analyze sentiment: {input}",
    "temperature": 0.7
  }
}

Tool Node

Purpose: Execute a specific function or tool

Use Cases: Data processing, API calls, calculations, validations

Available Tools:
  • text_analysis - Analyze text properties (word count, sentiment, etc.)
  • data_transform - Transform data structures
  • math_calculator - Perform calculations
  • data_validator - Validate data format and content
  • random_number - Generate random numbers
  • delay_simulator - Add delays (testing)
Properties:
Property Description Example
Tool Name Which tool to execute text_analysis
Parameters JSON object with tool parameters {"operation": "sentiment"}

Human Node

Purpose: Pause workflow for human approval or input

Use Cases: Approvals, reviews, manual data entry

Properties:
Property Description Example
Assigned To User who should approve admin, manager@example.com
Task Type Type of human task approval, review, input
Instructions What the human should do Please review and approve this report
Timeout Time limit in seconds 3600 (1 hour), 86400 (1 day)
Important: When a workflow reaches a Human node, it PAUSES until the assigned person completes the task.

Conditional Node

Purpose: Branch workflow based on conditions

Use Cases: Decision making, routing, filtering

Properties:
Property Description Example
Condition Python expression to evaluate result['score'] > 0.8
Tip: You can connect the conditional node to different paths based on true/false outcomes.

Parallel Node

Purpose: Execute multiple branches concurrently

Use Cases: Speed up processing, run independent tasks simultaneously

Properties:
Property Description Example
Branches List of node IDs to run in parallel ["analyze_a", "analyze_b"] 
                ┌──→ [Tool A] ──┐
[Start] → [Parallel]           → [End]
                └──→ [Tool B] ──┘

Workflow Examples

Example 1: Simple Text Analysis

Use Case: Analyze customer feedback sentiment

[Start] ───→ [LLM: Analyze Sentiment] ───→ [End]

Configuration:

  • LLM Node (analyze_sentiment)
    • Model: GPT-4
    • Prompt: "Analyze the sentiment of this customer feedback and classify as positive, negative, or neutral: {input}"
    • Temperature: 0.3 (focused)

Example 2: Report Generation with Approval

Use Case: Generate report, get manager approval, then send

[Start] → [LLM: Generate] → [Human: Approve] → [Tool: Send] → [End]

Configuration:

  • LLM Node (generate_report)
    • Model: GPT-4
    • Prompt: "Generate a weekly sales report based on: {input}"
  • Human Node (manager_approval)
    • Assigned To: manager@company.com
    • Task Type: approval
    • Instructions: "Please review the generated sales report and approve"
    • Timeout: 7200 (2 hours)
  • Tool Node (send_report)
    • Tool Name: send_email
    • Parameters: {"to": "team@company.com"}

Example 3: Conditional Routing

Use Case: Route high-priority issues differently

                        ┌──→ [High Priority Path] ──→ [Notify Manager] ──┐
[Start] → [Analyze] → [Conditional]                                       → [End]
                        └──→ [Normal Path] ──→ [Standard Process] ─────┘

Configuration:

  • LLM Node (analyze_priority)
    • Prompt: "Analyze this support ticket and assign a priority score (0-1): {input}"
  • Conditional Node (check_priority)
    • Condition: result['priority_score'] > 0.8

Example 4: Parallel Processing

Use Case: Run multiple analyses simultaneously

                    ┌──→ [Sentiment Analysis] ──┐
[Start] → [Parallel]    [Topic Extraction]     → [Combine] → [End]
                    └──→ [Language Detection] ──┘

Why Parallel? All three analyses can run at the same time, making the workflow 3x faster!

Example 5: Multi-Step AI Pipeline

Use Case: Research → Analyze → Summarize → Review

[Start] → [LLM: Research] → [Tool: Validate] → [LLM: Summarize] → [Human: Review] → [End]

Tips & Tricks

Toolbar Buttons Quick Reference

  • Save Workflow: Save to database for execution
  • Load: Open file dialog to load .json workflow
  • Clear: Remove all nodes from canvas (with confirmation)
  • Export JSON: Copy or download workflow as JSON file
  • Validate: Check workflow for errors before saving
  • View JSON: Preview the generated workflow definition
  • Zoom +/-: Zoom in/out (25% to 200%)
  • Help: Open this help guide in new tab

Layout Best Practices

  • Flow Left to Right: Place Start on the left, End on the right
  • Use Space: Don't crowd nodes together - leave breathing room
  • Avoid Crossings: Try to minimize connection lines crossing each other
  • Group Related Nodes: Keep nodes that work together close to each other
  • Align Nodes: Use the grid background to keep nodes aligned

Naming Conventions

  • Be Descriptive: Use names like "analyze_sentiment" not "node1"
  • Use Underscores: Separate words with underscores: "generate_report"
  • Action Verbs: Start with what the node does: "validate_data", "send_email"
  • Keep It Short: Aim for 2-3 words maximum

Common Mistakes to Avoid

  • ❌ Forgetting Start/End Nodes: Every workflow needs both!
  • ❌ Disconnected Nodes: Make sure all nodes are connected to the flow
  • ❌ Empty Configurations: Fill in all required properties
  • ❌ Missing Tool Selection: Tool nodes must have a tool selected
  • ❌ Complex Conditions: Keep conditional expressions simple and testable

Testing Your Workflow

Before Saving:
  1. Click Validate to check for errors
  2. Fix any red errors (must fix)
  3. Review yellow warnings (should fix)
  4. Click View JSON to see the generated definition
  5. Save when validation passes!

Prompt Template Variables

Use variables in your LLM prompts to reference workflow state:

Examples:
  "Analyze this text: {input}"
  "Previous result was: {previous_result}"
  "User requested: {user_query}"
  
Variables are wrapped in {curly_braces}

Performance Tips

  • Use Parallel Nodes: For independent tasks that can run simultaneously
  • Keep Prompts Focused: Shorter prompts = faster LLM responses
  • Set Appropriate Timeouts: Don't make humans wait forever
  • Cache Results: Use tool nodes to cache expensive operations

Troubleshooting

Connection Won't Create

Problem: Can't connect two nodes

Solutions:

  • Make sure you're dragging from OUTPUT (right) to INPUT (left)
  • Start nodes don't have input ports
  • End nodes don't have output ports
  • Try clicking the port and holding while dragging

Properties Won't Save

Problem: Changes to properties aren't saving

Solutions:

  • Click the "Save Properties" button after making changes
  • Check for red error messages in property fields
  • Make sure JSON in parameters field is valid

Validation Errors

Problem: Can't save due to validation errors

Solutions:

  • Click Validate to see specific error messages
  • Ensure you have exactly ONE Start node
  • Ensure you have at least ONE End node
  • Check that all Tool nodes have a tool selected
  • Check that all Conditional nodes have a condition

Workflow Not Executing

Problem: Workflow saved but won't execute

Solutions:

  • Check that all nodes are connected
  • Verify LLM prompts have proper variable syntax
  • Ensure tool parameters are valid JSON
  • Check execution logs for specific error messages
