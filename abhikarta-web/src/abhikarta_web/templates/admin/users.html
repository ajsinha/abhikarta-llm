Key Terms on This Page
User

A person who interacts with the system, with credentials and assigned roles.

Role

A named set of permissions (e.g., Admin, Developer, User) that determines access.

RBAC (Role-Based Access Control)

Security model where permissions are assigned to roles, and users are assigned roles.

Super Admin

Highest privilege role with full access to all system features.

Active/Inactive

User status indicating whether the account can be used for login.

HITL Reviewer

Role for users who review and approve human-in-the-loop tasks.
