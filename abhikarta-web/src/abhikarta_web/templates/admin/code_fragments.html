How to Use Code Fragments

Reference approved or published code fragments in your workflows using URI patterns:

  • db://<fragment-name> - Load from database (e.g., db://data-utils)
  • file://<path> - Load from filesystem (e.g., file:///opt/code/utils.py)
  • s3://<bucket>/<key> - Load from S3 (e.g., s3://my-bucket/code/helpers.py)

Key Terms on This Page
Code Fragment

A reusable Python code module that can be dynamically loaded as a tool.

Source URI

Location of code: db:// (database), file:// (filesystem), or s3:// (cloud).

Category

Classification of code fragments (utility, integration, data, etc.).

Entry Point

The main function that will be called when the tool is executed.

Dynamic Loading

Loading code at runtime without requiring application restart.

Tool Registration

Process of making a code fragment available in the tools registry.
