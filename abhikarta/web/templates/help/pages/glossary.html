Glossary of Terms

A comprehensive dictionary of terms, concepts, and acronyms used throughout the Abhikarta LLM platform.

Jump to: A B C D E F G H I J K L M N O P R S T U V W

A

Agent

An autonomous AI entity that can perceive its environment, make decisions, and take actions to achieve specific goals. In Abhikarta LLM, agents are configurable units that combine LLMs with tools and workflows to perform complex tasks.

Agent Type

The architectural pattern that defines how an agent reasons and acts. Common types include:

  • ReAct - Reasoning and Acting pattern where the agent iteratively reasons about actions and observes results
  • Chain of Thought (CoT) - Step-by-step reasoning through problems
  • Plan and Execute - Creates a complete plan before execution
  • Custom - User-defined agent logic
API (Application Programming Interface)

A set of protocols and tools that allow different software applications to communicate. Abhikarta LLM provides REST APIs for programmatic access to agents, workflows, and tools.

API Key

A unique identifier used to authenticate requests to an API. Required for accessing LLM providers like OpenAI, Anthropic, and others.

Audit Log

A chronological record of system activities, including user actions, agent executions, and configuration changes. Used for security monitoring and compliance.

B

Bedrock

Amazon Web Services' fully managed service for building generative AI applications. Provides access to foundation models from AI21 Labs, Anthropic, Cohere, Meta, and Amazon.

Bearer Token

An authentication credential passed in HTTP headers (Authorization: Bearer {token}) to access protected resources. Commonly used with OAuth 2.0 and API authentication.

C

Chain of Thought (CoT)

A prompting technique where the model is encouraged to show its reasoning process step-by-step before arriving at a final answer. Improves performance on complex reasoning tasks.

Code Fragment

A reusable piece of Python code that can be dynamically loaded and executed as a tool. Code fragments are stored in the database and can be managed through the admin interface.

Completion

The text generated by an LLM in response to a prompt. Also called "generation" or "response".

Context Window

The maximum amount of text (measured in tokens) that an LLM can process in a single request. Includes both input prompt and generated output. Different models have different context window sizes (e.g., 4K, 8K, 32K, 128K tokens).

CRUD

Create, Read, Update, Delete - the four basic operations for persistent data storage. Abhikarta LLM supports CRUD operations for agents, workflows, tools, and other entities.

D

DAG (Directed Acyclic Graph)

A graph data structure with directed edges and no cycles. Used in Abhikarta LLM to represent workflow execution order, ensuring tasks are executed in the correct sequence.

Dashboard

The main interface screen that provides an overview of system status, recent activities, and quick access to key functions.

E

Embedding

A numerical vector representation of text that captures semantic meaning. Used for similarity search, clustering, and retrieval-augmented generation (RAG).

Endpoint

A specific URL where an API can be accessed. For example, /api/agents is an endpoint for managing agents.

Execution

A single run of an agent or workflow. Each execution has a unique ID, status (pending, running, completed, failed), input parameters, and output results.

Execution Trace

A detailed log of all steps taken during an execution, including tool calls, LLM interactions, and intermediate results. Useful for debugging and optimization.

F

Few-Shot Learning

A technique where the model is given a few examples in the prompt to guide its behavior. Contrasted with zero-shot (no examples) and many-shot learning.

Fine-Tuning

The process of further training a pre-trained model on specific data to improve performance on particular tasks.

Function Calling

A capability where LLMs can generate structured output to call external functions or tools. The model determines which function to call and what arguments to pass.

G

GPT (Generative Pre-trained Transformer)

A family of large language models developed by OpenAI. The architecture is based on the Transformer neural network.

Guardrails

Safety mechanisms that prevent AI systems from producing harmful, biased, or inappropriate outputs. Can include content filters, output validators, and ethical guidelines.

H

Hallucination

When an LLM generates information that appears plausible but is factually incorrect or entirely fabricated. A known limitation of generative AI systems.

HITL (Human-in-the-Loop)

A workflow pattern where human oversight and intervention are integrated into automated processes. Humans review, approve, or modify AI decisions before they take effect. Essential for high-stakes applications requiring human judgment.

I

Inference

The process of using a trained model to generate predictions or outputs. In LLMs, inference refers to generating text completions.

Input Schema

A JSON Schema definition that describes the expected structure and types of input parameters for a tool or API endpoint.

J

JSON (JavaScript Object Notation)

A lightweight data interchange format. Used extensively in Abhikarta LLM for configuration files, API requests/responses, and tool schemas.

JSON Schema

A vocabulary for annotating and validating JSON documents. Used to define the structure of tool parameters, API payloads, and configuration objects.

K

KYC (Know Your Customer)

Regulatory requirements for financial institutions to verify customer identity. Abhikarta LLM includes pre-built banking agents for KYC verification workflows.

L

LangChain

A popular open-source framework for building applications with LLMs. Abhikarta LLM integrates with LangChain for agent orchestration and tool management.

LangGraph

A library for building stateful, multi-actor applications with LLMs. Used in Abhikarta LLM for complex workflow orchestration.

LLM (Large Language Model)

An AI model trained on vast amounts of text data that can understand and generate human-like text. Examples include GPT-4, Claude, Llama, and Gemini.

LLM Provider

A service that hosts and provides access to large language models via APIs. Supported providers in Abhikarta LLM include OpenAI, Anthropic, Google, AWS Bedrock, Azure, Mistral, Cohere, Groq, Together AI, HuggingFace, and Ollama.

M

Max Tokens

The maximum number of tokens the model should generate in its response. Controls output length and helps manage costs.

MCP (Model Context Protocol)

An open protocol for connecting AI assistants to external data sources and tools. MCP servers expose tools that can be dynamically discovered and integrated into Abhikarta LLM.

MCP Tool Server

A server that implements the Model Context Protocol, exposing one or more tools via a standardized API. Can be connected via HTTP/REST or stdio transport.

Model

A specific version of an LLM offered by a provider. For example, "gpt-4o", "claude-3-sonnet", or "llama-3.2-70b".

N

Node

A single unit of work in a DAG workflow. Nodes can represent LLM calls, tool executions, conditionals, or other operations.

O

Ollama

An open-source tool for running large language models locally. Supports models like Llama, Mistral, and others without requiring cloud API calls.

Orchestration

The coordination of multiple components (agents, tools, workflows) to accomplish complex tasks. Abhikarta LLM provides orchestration capabilities for building sophisticated AI applications.

P

Parameter

An input variable for a tool, function, or API. Parameters have names, types, descriptions, and may be required or optional.

Pre-built Tool

A ready-to-use tool included with Abhikarta LLM. Categories include Common (date/time, math, text), Banking (account, loan, fraud), Integration (web search, database), and General (file, calculation).

Prompt

The input text sent to an LLM to elicit a response. Includes user messages, system instructions, and any context or examples.

Prompt Engineering

The practice of designing and optimizing prompts to get desired outputs from LLMs. Includes techniques like few-shot learning, chain of thought, and structured output formatting.

R

RAG (Retrieval-Augmented Generation)

A technique that combines information retrieval with text generation. The model retrieves relevant documents from a knowledge base and uses them to generate more accurate, grounded responses.

RBAC (Role-Based Access Control)

A security model where permissions are assigned to roles, and users are assigned to roles. Abhikarta LLM implements RBAC with roles including Super Admin, Domain Admin, Agent Developer, Agent Publisher, HITL Reviewer, Agent User, and Viewer.

ReAct (Reasoning and Acting)

An agent architecture pattern where the model alternates between reasoning about the task and taking actions. The agent observes action results and reasons about next steps iteratively.

Registry

A centralized catalog of available components. Abhikarta LLM maintains registries for tools, agents, workflows, and LLM providers.

REST API

Representational State Transfer - an architectural style for web APIs. Uses HTTP methods (GET, POST, PUT, DELETE) to perform operations on resources identified by URLs.

Role

A named set of permissions that can be assigned to users. Determines what actions a user can perform within the system.

S

Schema

A formal definition of data structure. In Abhikarta LLM, schemas define tool parameters, API payloads, and database tables.

STDIO Transport

Standard Input/Output - a method of communication where data is exchanged through standard input and output streams. Used by some MCP servers for local tool execution.

System Prompt

Instructions provided to an LLM that define its role, behavior, and constraints. Typically set at the beginning of a conversation and influences all subsequent responses.

T

Temperature

A parameter that controls the randomness of LLM outputs. Lower values (0.0-0.3) produce more deterministic, focused responses. Higher values (0.7-1.0) produce more creative, varied outputs.

Token

The basic unit of text that LLMs process. Tokens can be words, parts of words, or characters. A rough estimate is 1 token ≈ 4 characters or ¾ of a word in English.

Tool

A function or capability that an agent can use to perform actions. Tools have names, descriptions, and parameter schemas. Examples include web search, database queries, calculations, and API calls.

Tool Call

An invocation of a tool by an agent. Includes the tool name and arguments. The tool executes and returns results that the agent can use for further reasoning.

Top-P (Nucleus Sampling)

A parameter that limits token selection to those with cumulative probability above a threshold. Alternative to temperature for controlling output diversity.

Trace

A detailed record of all operations performed during an execution. Includes timestamps, inputs, outputs, and any errors encountered.

Transformer

A neural network architecture that uses self-attention mechanisms. The foundation of modern LLMs including GPT, Claude, and Llama.

U

User

A person who interacts with the Abhikarta LLM system. Users have credentials (user ID, password), roles, and permissions.

V

Vector Database

A database optimized for storing and searching vector embeddings. Used for semantic search and RAG applications.

W

Workflow

A defined sequence of steps that accomplish a specific task. In Abhikarta LLM, workflows are represented as DAGs with nodes (operations) and edges (dependencies).

Workflow Designer

A visual interface for creating and editing workflows. Allows drag-and-drop node placement, connection drawing, and configuration.
